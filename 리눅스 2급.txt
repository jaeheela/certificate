## 프로세스
- 관련 프로세스가 메모리에 항상 상주하는 것이 아닌, 클라이언트의 서비스 요청이 들어오면 관련 프로세스를 실행하고 서비스가 즁료되면 함께 종료됨 : inetd
- 프로세스 A가 프로세스 B를 실행시킨 후, B PPID 조회하니 A의 PID 임 : fork
- 하나의 프로세스가 새로운  프로세스를 생성할 때 새로운 프로세스를 위해 메모리를 할당받아 복사본 형태로 실행함 : fork
- 요청이 빈번하고 빠른 서비스를 제공해야하는 경우 데몬 프로세스의 방식 : standalone
- PID 1222번 강제 종료 : kill -9 1222
- 우선순위 변경 명령으로 설정가능한 NI 범위값 : -20 ~ 19

# exec
- 하나의 프로세스가 새 프로세스를 생성할 때 원래의 프로세스를 새 프로세스로 대체하는 형태
- 호출한 프로세스의 메모리에 새로운 프로세스의 코드를 덮어씌움
- 다른 프로세스를 실행하기 위한 시스템 호출 방법
- 특정 프로세스가 새로운 프로세스를 발생시킬 때 프로세스가 증가하지 않는 방식
- exec ps -l

- CentOS6의 최초 프로세스명 : init
- CentOS7의 최초 프로세스명  : systemd
- 리눅스가 부팅을 시작하면 커널이 최초의 프로세스인 init 프로세스를 발생시킴, 할당 PID : 1
- init 프로세스는 시스템 운영에 필요한 데몬 등을 비롯하여 다른 프로세스들은  fork 방식으로 생성함

- 실행중인 모든 프로세서 확인 : ps aux
- 프로세스 상태 출력 (System V 계열에서 모든 프로세스를 출력) : ps -A

- 현재 로그인에서 사용 중인 셀의 우선 순위 항목값인 NI, PRI 확인 : ps
- 프로세스의 우선순위와 관련된 항목 : PRI, NI
- PRI : 커널에서 참고하여 운영체제가 실제 참고하는 항목
- NI : 사용자가 변경하는 항목으로 NI 값이 PRI 값에 반영됨
- 우선순위가 인위적으로 높아진 상태를 의미하는 프로세스 상태 코드 값 : <
- NI 값 중 우선수위가 가장 낮게 할당된 값 : 10 < -20,20,0

## cron
- cron으로 매주 월~금까지 오후 12시에 백업 스크립트 실행 : 0 12 * * 1-5 /etc/backup.sh
- cron을 이용해 해당 스크립트를 매주 토,일 오전 4시1분에 주기적으로 실행 : 1 4 * * 0,6 /etc/backup.sh
- 백업 스크립트가 30분 주기로 실행되도록 crontab에 설정 : */30* * * * /etc/backup.sh
- cron을 이용해 해당 스크립트를 5분 주기로 실행 : * / 5 * * * * /etc/heartb eat.sh
- 백업 스크립트가 일주일에 1회만 실행되도록 crontab에 설정 : 1 1 * * 5 /etc/backup.sh

## bash
- os 라는 셀 변수에 linux 라는 값을 선언 : os=linux
- bash 프로세스의 PID 1222일 때 nice 명령어 사용법 : nice bash (=> killall, nice : 인자값으로 프로세스명)
- nice -10 bash : PID가 1222인 bash 프로세스의 우선순위값이 0에서 NI 값을 10으로 변경
- 사용중인 bash 프로세스의 PID 1222일 때 renice 명령어 사용법 : renice 1 1222
- ~/bash_logout : kaituser는 로그아웃을 할 때 자동으로 생성되는 .bak 로 끝나는 파일을 삭제하려고 함
- ~/.bashrc : 특정 사용자의 ~/bashrc 파일에 alias 와 function 설정 가능
- nohup : 오랜 시간 작업이 소요되는 백업 명령, 작업 중인 터미널이 닫혀도 계속 실행 가능
- renice : PID로 실행 중인 프로세스의 우선순위 변경

- 메일 관련 프로토콜: pop3(110), smtp(25), imap(143)
- smtp : 메일 서버 간 메시지 교환할 때 사용(전자메일)

DNS 포트번호 : 53

##IRC
- 실시간 채팅 프로토콜
- 여러 사용자가 모여 대화 가능한 서비스, 개인 가의 대화와 파일 전송기능도 제공

루프백 ip 주소 : 127.0.0.1


## 리눅스 파일 역할
- 시스템 점검을 위해 외부 네트워크와 연결을 차단하려고 함 : /etc/sysconfig/network
네트워크 인터페이스 환경 설정과 관련된 파일들이 저장된 디렉터리 : /etc/sysconfig/network-scripts
- 네임서버가 기록된 파일 : /etc/resolv.conf

## 삼바
- 삼바가 처음 등장했을 때 - SMB(server message block) 프로토콜 사용, 최근 - CIFS 사용
- 삼바 서비스와 관련 깊은 프로토콜 : CIFS
- 윈도우와 리눅스 파일 공유 : 삼바
- 리눅스와 리눅스 파일 공유 : NFS
- NFS 서버 사용 시에 반드시 구동해야할 데몬 : RPCBIND

## Docker
- 서버운영에 필요한 프로그램을 이미지로 만들어 프로세스처럼 동작
- 실행되는 이미지 = 컨테이너
- 컨테이너 내부에 접속 가능
- 컨테이너는 이미지로 저장 가능
- 컨테이너는 외부저장소를 통해 배포 가능
- 프로세스 격리 기술로 컨테이너 실행하고 관리하는 오픈 소스 프로그램

## 쿠버네티스
- 컨테이너화된 애플리케이션 배포, 관리, 확장 프로그램

# 클라우드서비스
- FaaS(Function as a Service) : 클라우드 컴퓨팅 서비스 모델 중 하나, 사용자가 프로젝트나 애플리케이션 기능을 서비스 형태로 등록해 특정 이벤트 발생 시에만 실행되고 작업 완료 시 종료됨

## 리눅스 가상화 기술(virtualbox)
- 인텔의 하드웨어 가상화 VT-x와 AMD의 AMD-V를 기반으로 전가상화를 지원
- VMware 에서 생성한 가상 머신의 파일 형식 : VMDK

## 리눅스가 사용하는 클러스터
- 고가용성 클러스터(HA) : 메인 이상 시 백업이 서비스 이어받음, 지속적인 서비스 제공이 목적
- 고계산용 클러스터(HPC)
- 부하분산 클러스터(LVS)
- 베어울프 클러스터 : 고성능의 계산 능력을 제공하기 위한 목적, 과학 계산용, 슈퍼컴

Tizen : 인텔과 삼성을 주축으로 리눅스 재단, MeeGo 개발자들이 합류하여 만든 리눅스 커널 기반의 모발일 운영체제
WebOS : 리눅스 커널에서 구동되는 모바일 운영체제, Palm OS를 계승한 소프트웨어, 현재 LG가 주도 개발

XEN : 반가상화를 지원하는 서버 가상화 기술
Ansible : 소프트웨어 프로비저닝, 구성관리, 배포프로그램으로 유닉스 및 리눅스, 윈도우 운영체제에서 사용 가능한 공개 프로그램, 2015 레드햇에 인수되어 관리
